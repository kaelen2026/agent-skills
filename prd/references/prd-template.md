# PRD Template

> 用于 `prd` 技能 Phase 1 的 PRD 模板。

```markdown
## 问题描述

{用非技术语言描述用户痛点。一段话，任何人都能理解。}

## 目标

{这个功能要达成什么？用 1-3 个可衡量的目标。}

## 目标用户

{谁会使用这个功能？列出角色和使用场景。}

## 用户故事

### Must Have

- [ ] 作为 {角色}，我想要 {行为}，以便 {价值}
  - 前置条件: {条件}
  - 期望结果: {详细描述用户看到/体验到的}
  - 边界情况: {异常情况及处理方式}

### Should Have

- [ ] ...

### Could Have

- [ ] ...

### Won't Have (本次不做)

- ...

## 模块设计

### 模块 1: {名称}

- **接口**: {输入 → 输出，尽量简单}
- **隐藏复杂性**: {模块内部处理的复杂逻辑}
- **依赖**: {依赖的其他模块}

### 模块 2: {名称}

- **接口**: ...
- **隐藏复杂性**: ...
- **依赖**: ...

## 架构决策

| 决策 | 选项 | 选择 | 理由 |
|------|------|------|------|
| {决策点} | A: ... / B: ... | {选择} | {为什么} |

## Schema 变更

{新增/修改的数据模型。只描述字段和关系，不含具体 SQL。}

## API 契约

{新增/修改的 API 端点。包含路径、方法、请求/响应结构、错误码。}

## 验收标准

- [ ] {描述可观察的外部行为，非实现细节}
- [ ] {用户执行 X → 系统响应 Y}
- [ ] {错误场景: 当 Z 发生时 → 用户看到 W}

## 未解决问题

- {需要进一步讨论的开放问题}

## 参考

- {相关文档、设计稿、竞品链接}
```
